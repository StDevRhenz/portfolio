ITEM 25
PHP Class — Student
Create a PHP class named Student with properties name and course, a constructor, and a displayInfo() method.
classconstructor$thismethodpublic
PHP
<?php
// Class = blueprint o template ng isang object
class Student {

    // Properties = mga variable sa loob ng class
    // "public" = pwede ma-access kahit saan
    public $name;
    public $course;

    // Constructor = awtomatikong tinatawag kapag gumawa ng bagong object
    // Dito natin ini-initialize ang mga values ng properties
    public function __construct($name, $course) {
        // $this = tumutukoy sa kasalukuyang object
        $this->name = $name;
        $this->course = $course;
    }

    // Method para ipakita ang details ng student
    public function displayInfo() {
        echo "Name: " . $this->name . ", Course: " . $this->course;
    }
}

// Gumawa ng bagong Student object - tinatawag ang constructor
$s = new Student("Ana", "BSIT");

// Tawagin ang displayInfo() method
$s->displayInfo(); // Output: Name: Ana, Course: BSIT
?>
ITEM 26
Inheritance — Person & Student
Write a PHP program demonstrating inheritance: parent class Person, child class Student that extends Person, display the inherited name.
extendsinheritanceparent classchild class
PHP
<?php
// Parent class = ang "nagbibigay" ng properties at methods
class Person {
    public $name;

    public function __construct($name) {
        $this->name = $name;
    }
}

// Child class - "extends" = nag-iinherit ng lahat mula sa Person
// Lahat ng properties at methods ng Person = available sa Student
class Student extends Person {
    // Walang laman - ginagamit na lang ang inherited na properties
}

// Gumagamit ng Person's constructor kahit Student ang ginawa
$s = new Student("Juan");

// Ang "name" ay galing sa Person, pero accessible sa Student
// Ito ang INHERITANCE — paggamit ng properties ng parent
echo $s->name; // Output: Juan
?>
ITEM 27
Static — Counter Class
Create a PHP class Counter with a static property $count = 0 and a static method increment() that increases the count.
staticself::ClassName::no instantiation
PHP
<?php
class Counter {

    // Static property = isa lang para sa LAHAT ng Counter objects
    // Hindi kailangan gumawa ng object para ma-access ito
    public static $count = 0;

    // Static method = tinatawag gamit ang ClassName:: (walang new)
    public static function increment() {
        // "self::" = tumutukoy sa sariling class (Counter)
        self::$count++;
    }
}

// Tinatawag ang static method nang walang "new Counter()"
Counter::increment(); // count = 1
Counter::increment(); // count = 2

// I-access ang static property gamit din ang "::"
echo Counter::$count; // Output: 2
?>
ITEM 28
Trait — Logger
Write a PHP example demonstrating the use of a trait named Logger with a method log($msg).
traitusecode reusemultiple classes
PHP
<?php
// Trait = "add-on" na pwedeng gamitin ng iba't ibang class
// Solusyon sa single-inheritance limitation ng PHP
trait Logger {

    // Method na kasama sa trait
    // Pwedeng gamitin ng kahit anong class na nag-"use" ng Logger
    public function log($msg) {
        echo "Log: " . $msg;
    }
}

class App {
    // "use" keyword = kinokopya ang mga methods ng trait dito
    // Parang "kinukuha" ang log() method mula sa Logger
    use Logger;
}

$app = new App();

// Kahit ang log() ay galing sa trait, pwede nating tawagin ito
$app->log("Hello!"); // Output: Log: Hello!
?>
ITEM 29
Abstract Class — Payment & GCash
Create an abstract class Payment with an abstract method processPayment(). Create a child class GCash that implements the method.
abstract classabstract methodextendsimplementation
PHP
<?php
// Abstract class = template, HINDI pwedeng gumawa ng direct object dito
// Nagse-set ng rules na DAPAT sundin ng child classes
abstract class Payment {

    // Abstract method = walang body/implementation dito
    // OBLIGADO ang child class na mag-implement nito
    abstract public function processPayment();
}

// Child class - nag-e-extend at nag-iimplement ng abstract method
class GCash extends Payment {

    // Kailangan TALAGANG ilagay ito — kung wala, mag-eerror ang PHP
    public function processPayment() {
        echo "Processing GCash payment.";
    }
}

// "new Payment()" = ERROR! Abstract class hindi pwedeng i-instantiate
// Gamitin ang child class:
$g = new GCash();
$g->processPayment(); // Output: Processing GCash payment.
?>
